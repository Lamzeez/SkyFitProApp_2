4. Technical Requirements
A. Secure Authentication (M1 & M3)
- Custom Registration: Users must provide Email, Password (Strong Regex), Create user profile like (Full Name, Age, Weight, Profile Picture, and other relative information all are updatable)
- Google SSO: "Sign in with Google" button must work.
B. Biometric Security (M2)
- Toggle: In ProfileView, add a switch: "Enable Biometric Login".
- Logic: If enabled, the next app launch prompts for Fingerprint/FaceID instead of password.
- Fallback: If biometrics fail 3 times, force password entry.
- Inactivity Timer:
  - Create a SessionManager class.
  - Use a Timer that resets on user interaction (tap/scroll).
  - Logic: if (inactiveTime > 5 mins) -> Navigator.pushReplacement(LoginScreen).
C. Personalized Health Logic (M3 & M5)
- API: Fetch data from OpenWeatherMap (Temp, Humidity, Weather Condition).
- Logic Engine: This is only a suggested activities based on the following matrix you can improved this by research:

5. Deployment Instructions (GCP & Firebase)
Step 1: Firebase Setup
- Create a Firebase Project and enable Authentication (Email/Password & Google) and Firestore.
- Web Config: For Flutter Web, use dart-define to inject firebaseConfig during the Docker build.

Step 2: Dockerfile (Multi-Stage)
Dockerfile:
# Stage 1: Build
FROM debian:latest AS build-env
# ... install flutter ...
RUN flutter build web --release --dart-define=API_KEY=SECRET_KEY
# Stage 2: Serve
FROM nginx:alpine
COPY --from=build-env /app/build/web /usr/share/nginx/html
EXPOSE 8080

Step 3: CI/CD (Cloud Build) Ensure cloudbuild.yaml handles the build arguments for your keys.
